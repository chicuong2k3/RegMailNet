# Blazor Hybrid UI for RegMailNet — Design Spec

## Overview

Add a MAUI Blazor Hybrid desktop app (`RegMailNet.Ui`) to the RegMailNet solution, providing a GUI for creating email accounts, managing proxies, configuring API keys, and viewing creation history.

## Architecture

- **New project**: `src/RegMailNet.Ui/RegMailNet.Ui.csproj` — .NET 10 MAUI Blazor Hybrid
- **Template base**: Copied from `chicuong2k3/BlazorHybridTemplate`
- **References**: `RegMailNet` (project ref), `BlazorBlueprint.Components`, `BlazorBlueprint.Icons.Lucide`
- **UI framework**: BlazorBlueprint.Components (Tailwind-style `BbCard`, `BbButton`, `BbAlert`, etc.)
- **Layout**: Sidebar navigation with 5 pages

## Pages

### 1. Dashboard (`/`)
- Stats cards: total accounts created, counts per provider (Outlook/Gmail/Yahoo)
- Service status grid: connected/missing for each captcha and SMS service
- Recent activity list: last 5 created accounts with email, provider, timestamp

### 2. Create Account (`/create`)
- Provider selection cards: Outlook, Gmail, Yahoo (click to select)
- Options panel: browser dropdown (Firefox/Chrome/Undetected Chrome), proxy toggle
- Single "Create" button — all account fields auto-generated by RegMailNet's `DataGenerator`
- Progress indicator (toast/snackbar) during creation
- Result displayed on completion: email, password, provider

### 3. History (`/history`)
- Table: email, password, provider, date created, status (success/failed)
- Filter by provider
- Data stored in-memory (list on `AccountHistoryService`)

### 4. Proxies (`/proxies`)
- List of configured proxies with remove button
- Add proxy input field
- Auto-proxy toggle (placeholder — not yet implemented in library)

### 5. Settings (`/settings`)
- Captcha API keys: Capsolver key, Nopecha key
- SMS API keys: SmsPool (token), 5Sim (token), GetsmsCode (user + token)
- Default browser selection
- Default SMS service selection

## Data Flow

1. `RegMailNetManager` registered as a singleton via DI
2. Account creation calls `RegMailNetManager.CreateOutlookAccount()` (etc.) on a background thread
3. Results stored in `AccountHistoryService` (in-memory list)
4. Settings read from/written to `appsettings.json` via `IOptions<T>` pattern

## State Management

- `AccountHistoryService`: singleton, holds `List<AccountCreationResult>` in memory
- `SettingsService`: reads/writes `appsettings.json` for API keys and defaults
- Component state via Blazor's built-in `@bind` and `StateHasChanged`

```
  Components/
    Layout/

    index.html               — from template

```

## Non-Goals

- No database persistence (in-memory history only)
- No user authentication
- No email reading/verification after creation
- No real-time progress streaming from Selenium (blocking call with loading indicator)
