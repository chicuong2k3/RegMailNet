# RegMailNet API Project Design

## Purpose

Add a FastEndpoints-based ASP.NET Core API project to expose RegMailNet's account creation operations as HTTP endpoints. This makes it easy to test account creation workflows via curl, Postman, or any HTTP client without needing the MAUI desktop UI.

## Technology

- **Framework**: ASP.NET Core 9 + FastEndpoints
- **Pattern**: REPR (Request-Endpoint-Response) via FastEndpoints
- **Validation**: FluentValidation (bundled with FastEndpoints)
- **Docs**: Swagger/OpenAPI (auto-generated by FastEndpoints)

## Project Structure

```
src/RegMailNet.Api/
  ├── Program.cs
  ├── Endpoints/
  │   ├── CreateGmailAccount.cs
  │   ├── CreateOutlookAccount.cs
  │   ├── CreateYahooAccount.cs
  │   └── HealthCheck.cs
  ├── Requests/
  │   └── CreateAccountRequest.cs
  ├── Responses/
  │   └── AccountCreatedResponse.cs
  └── RegMailNet.Api.csproj
```

## Endpoints

### POST /api/accounts/gmail
Creates a Gmail account.

### POST /api/accounts/outlook
Creates an Outlook account.

### POST /api/accounts/yahoo
Creates a Yahoo account.

### GET /api/health
Health check endpoint returning service status.

## Request/Response DTOs

**CreateAccountRequest** (optional overrides, all fields optional since RegMailNetManager generates defaults):
- `string? Username` - Override generated username
- `string? Password` - Override generated password
- `string? FirstName` - Override generated first name
- `string? LastName` - Override generated last name
- `string? Proxy` - Proxy to use for this request

**AccountCreatedResponse**:
- `string Email` - Created email address
- `string Password` - Account password
- `string Provider` - Email provider name
- `DateTime CreatedAt` - Timestamp

## Wiring

- Program.cs calls `services.AddRegMailNet()` to register all core services
- Resolves `RegMailNetManager` from DI for each endpoint
- Configuration via appsettings.json (same config format as existing library)

## Dependencies

- FastEndpoints (includes Swagger support)
- Project reference to src/RegMailNet
